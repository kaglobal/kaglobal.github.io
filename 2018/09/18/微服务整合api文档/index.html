    spring多模块 整合springcloud gateway

    • 比如目前有2个微服务,一个网关,现在能够通过网关能够直接访问到对应微服务，结构如下:
    服务名 服务用途
    mall-gateway 网关服务
    mall-admin 后台admin服务
    mall-order 订单服务
    mall-gateway网关pom依赖
    • 注意springcloud gateway 实现是webflux 与start -web有冲突 所以这里需要将spring-boot-starter-web的pom依赖删除,否则启动报错
      mall-gateway网关application.yml
    mall-admin admin服务
    • mall-admin服务就正常的springboot web配置就可以了,当前这里由于我们没连数据库测试所以启动类排除下jdbc数据源连接避免自动初始化数据库连接池
    • 写一个测试的controller TestController

    整合完成,浏览器访问:
    http://localhost:8082/mall-admin/test
    http://localhost:8082/mall-admin/test
    分别返回test,test1

    springcloud gateway 整合 knife4j

    • 目的:为了实现多木块==模块服务可以通过gateway网关统一入口查看api文档
    • knife4j官网整合文档地址:https://doc.xiaominfo.com/guide/ui-front-gateway.html#eureka%E6%B3%A8%E5%86%8C%E4%B8%AD%E5%BF%83
    • 目前官网也有详细说明,我也是借鉴官网完成搭建,但是其中@EnableSwaggerBootstrapUi的注解的依赖一直找不到,后面使用@EnableKnife4j完美解决

    总结遇到的坑

    • 网关mall-gateway application.yml 配置的SwaggerHeaderFilter 为网关整合knife4j 这个类需要添加@Component启动自动装配,否则启动找不到这个filter
    • 网关mall-gateway pom依赖增加ui的依赖
    • mall-admin服务SwaggerConfiguration类 @EnableSwaggerBootstrapUi 改为@EnableKnife4j
